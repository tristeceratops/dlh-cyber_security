# The Misconfiguration Findings

## Introduction

### Goal
Analyze vulnerabilities that have no CVE identifier and understand why they are equally dangerous.

### Context
he scan report contains findings marked as "Misconfiguration" with no CVE. No CVE means no CVSS score, no NVD page, no Exploit-DB entry. It also means most automated prioritization tools will ignore them. That is exactly the problem.

The MongoDB Ransomware Wave of 2017 affected 28,000 databases. Not one had a CVE. Every single compromise was a misconfiguration: databases exposed to the internet with no authentication. The Capital One breach of 2019 that exposed 100 million records was a misconfiguration. Not a software bug. A misconfigured AWS WAF rule.

| Finding 010 | 10.10.3.40-46 (BD Alaris infusion pumps) | Seven infusion pumps expose web management interfaces with unchanged default credentials (`admin/admin`) and lack proper network isolation. | No CVE is assigned for the default credentials because the device software is not necessarily defective. Default credentials may be more dangerous because exploitation requires no technical vulnerability and is immediately usable. |
| Finding 015 | 10.10.2.41 (NAS-01 Backup Storage) | Synology DSM management interface is accessible from the entire internal network and backup data is stored unencrypted. | No CVE exists because the NAS is not malfunctioning. The issue is incorrect access control and insecure backup storage practices. | **Medium** — Compromise of backups could allow ransomware impact, data loss, or exposure of sensitive recovery data. | **1x00 T5 – Control gap:** Missing backup security controls and administrative access restrictions. **1x00 T7 – Network scan finding:** Management interface exposure was detected. | **CVE-2021-43798 (Grafana path traversal, CVSS 7.5):** Although Grafana has a software vulnerability, an exposed backup management interface can create similar confidentiality risks by allowing attackers access to sensitive organizational data. |

The statement "Our CVE scan shows nothing critical, we are secure" provides dangerous false assurance because CVE scanning only identifies known software vulnerabilities and does not measure insecure configurations, excessive permissions, weak authentication, exposed services, missing segmentation, or unsafe operational practices. This report demonstrates that several critical risks, including unrestricted database access, insecure medical devices, and exposed administrative interfaces, have no CVE number but could still lead to major breaches, patient data exposure, or operational disruption. Security posture must therefore consider both technical vulnerabilities and configuration/control weaknesses.
